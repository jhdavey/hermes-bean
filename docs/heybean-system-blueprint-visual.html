HeyBean System Architecture Blueprint

Canonical Laravel backend and runtime owner, Flutter + browser command-center clients, workspace-scoped Bean agency, Laravel-owned tools/memory/queues, OpenAI-compatible model providers, and external integrations.

Client responsibility

  • • Flutter and browser render the command center.
  • • Clients call Laravel API; they do not own durable business state.
  • • Voice/WebRTC orchestration happens client-side, but privileged work returns to Laravel.

Runtime responsibility

  • • Laravel binds HermesRuntimeService to HermesToolRuntimeService.
  • • Runtime chooses full, app_crud, or read_lookup routes.
  • • CRUD planner is a fast path; native tools remain Laravel-executed.

State responsibility

  • • Workspace is the main data and agency boundary.
  • • Sessions, messages, runs, events, memory, approvals, and blockers are persisted.
  • • dashboard_changes lets clients refresh mutations caused by Bean or integrations.

Realtime voice split

  • • Realtime model gives low-latency spoken responses from snapshot context.
  • • Durable app changes queue assistant runs.
  • • Only queue_bean_work and cancel_bean_work are exposed to the realtime model.
